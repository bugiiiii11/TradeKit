**Description:** Security auditor for Claude Code skills. Scans a skill for data exfiltration, prompt injection, destructive operations, persistence backdoors, supply chain attacks, obfuscation, and scope creep, then produces a verdict and (on approval) installs the skill with a full analysis document.
**Source:** pasted by user

**Installed:** 2026-04-10

## What This Skill Does

When invoked with a file path, pasted content, or URL:

1. **Ingests** the target skill via Read or WebFetch.
2. **Scans** the skill across 7 security categories (exfiltration, prompt injection, destructive ops, persistence, supply chain, obfuscation, scope creep) and assigns CLEAN/SUSPICIOUS/DANGEROUS per category.
3. **Decodes** any base64/hex/URL-encoded content it finds and re-scans the decoded payload.
4. **Runs a quality checklist** against best practices (clear description, reads-before-writes, confirmation prompts, etc.).
5. **Detects overlap** with already-installed skills in `.claude/commands/`.
6. **Presents a report** with an overall verdict (SAFE / CAUTION / DANGEROUS) plus per-category findings with exact quotes.
7. **Waits for explicit user approval** before writing anything.
8. On approval, writes two files:
   - `.claude/commands/<skill-name>.md` (the skill itself)
   - `.claude/commands/<skill-name>_analysis.md` (the analysis document)
9. Has a "reorganize mode" that back-fills analysis documents for any already-installed skills that don't have one.

It never executes code from the skill being analyzed.

## Skill Architecture

- **Inputs:** file path, pasted content, or URL to a Claude Code skill
- **Processing:** 8-step workflow (ingest -> scan -> decode -> quality check -> overlap check -> report -> await approval -> install)
- **Outputs:**
  - Console report with verdict and findings
  - On approval: `<skill-name>.md` + `<skill-name>_analysis.md` in `.claude/commands/`
- **Dependencies:** Read, Write, Glob, WebFetch (all built-in Claude Code tools)

## Possible Improvements

- **Idea:** Compute a SHA-256 hash of the installed skill file and record it in the analysis document.
  **Why:** Lets the user detect tampering later. If a skill is modified after install, a quick hash check catches it.

- **Idea:** Support a "dry-run" flag that runs the full scan but exits before any writes, even on SAFE verdict.
  **Why:** Useful for reviewing a skill without committing to install.

- **Idea:** Add a "reference hash pin" mode where the user can lock a known-good version of a skill and get warnings if a re-install would change it.
  **Why:** Prevents silent updates to skills that introduce new risks.
